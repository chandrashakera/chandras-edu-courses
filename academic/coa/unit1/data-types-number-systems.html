Data Types & Number Systems

⏱ 15 min read 📝 Quiz inside 📖 Mano Ch. 1

Every computer, from a microcontroller to a data-center GPU, ultimately does one thing: it moves 0s and 1s around. Before any of the interesting computer-organization topics — arithmetic, instruction cycles, memory hierarchy — make sense, you need a solid grip on how numbers, letters, and symbols get turned into those 0s and 1s. That's what this page covers.

What Data Does a Computer Store?

Registers inside a digital computer hold one of two things: control information (bits that sequence the operations the hardware should perform) or data — the actual numbers and coded information being operated on. Data falls into three broad categories:

  • Numbers used in arithmetic computations
  • Letters of the alphabet used in text and data processing
  • Other discrete symbols — punctuation, control characters, and so on

With one exception — numbers already in pure binary — every one of these is represented inside the machine in binary-coded form: some agreed-upon mapping from a pattern of 0s and 1s back to the symbol it stands for.

Common Number Systems

A number system of radix (or base) r uses r distinct digit symbols, numbered 0 through r − 1, to represent quantities.

SystemRadix (r)Digit SymbolsUsed for
Decimal100 – 9Human input/output
Binary20, 1Hardware storage
Octal80 – 7Shorthand for binary
Hexadecimal160–9, A–FCompact binary shorthand

Positional Notation

Every number system uses the same rule — each digit's contribution to the total value is that digit multiplied by the radix raised to the power of its position:

value = Σ (digitᵢ × rⁱ) — where i is the position index, 0 at the rightmost digit (negative for fractional digits).

Converting Between Number Systems

Any Radix → Decimal

Expand the positional-notation formula and add up the terms:

(101101)₂ = 1×2⁵+0×2⁴+1×2³+1×2²+0×2¹+1×2⁰ = 32+8+4+1 = (45)₁₀
(736.4)₈  = 7×8²+3×8¹+6×8⁰+4×8⁻¹ = 448+24+6+0.5 = (478.5)₁₀
(F3)₁₆   = 15×16¹+3×16⁰ = 240+3 = (243)₁₀

Decimal → Any Radix

  • Integer part: divide repeatedly by r; read remainders from last to first (bottom to top).
  • Fractional part: multiply repeatedly by r; read carry digits from first to last (top to bottom). May not terminate.

Worked Example — Convert (41.6875)₁₀ to Binary

20 ÷ 2 = 1000.375 × 2 = 0.750
10 ÷ 2 = 500.75 × 2 = 1.51
5 ÷ 2 = 210.5 × 2 = 1.01

(41.6875)₁₀ = (101001.1011)₂

Worked Example — Convert (41.6875)₁₀ to Octal

Integer: 41 ÷ 8 = 5 rem 1; 5 ÷ 8 = 0 rem 5 → integer = 51
Fraction: 0.6875 × 8 = 5.5 → 5; 0.5 × 8 = 4.0 → 4
Result: (51.54)₈

Worked Example — Convert (41.6875)₁₀ to Hexadecimal

Integer: 41 ÷ 16 = 2 rem 9; 2 ÷ 16 = 0 rem 2 → integer = 29
Fraction: 0.6875 × 16 = 11.0 → B (terminates)
Result: (29.B)₁₆

Fast Conversion via Bit Grouping

Because 8 = 2³ and 16 = 2⁴, you can convert binary to octal/hex by simply grouping the bits — no arithmetic needed. Each octal digit = 3 bits; each hex digit = 4 bits.

Fig 2: Bit grouping — binary ↔ octal (3 bits) and hexadecimal (4 bits)
⚠️ Always pad with leading zeros to complete the outermost group before grouping — forgetting this shifts every digit in the result.

Common Pitfalls

  • Fractional conversions may not terminate. Converting 0.1 (decimal) to binary never ends — this is why 0.1 + 0.2 ≠ 0.3 in every programming language.
  • Pad before grouping. Forgetting to pad to a full multiple of 3 (octal) or 4 (hex) bits shifts every digit.
  • The radix fixes the symbol set. Digit 9 doesn't exist in octal — if you see it, the number isn't valid octal.

Applications

  • Computing: memory addresses and machine instructions are binary; hex is the practical shorthand engineers read.
  • Color codes: #FF0000 is pure red — each hex pair gives one RGB channel a value from 0–255.
  • Networking: IP addresses, MAC addresses, and checksums use hex for compactness.
  • Cryptography: keys and hashes are displayed in hex mapping cleanly to the underlying binary data.
