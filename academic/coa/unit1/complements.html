Complements

Learning Objectives

  • Define (r−1)'s complement and r's complement for any base
  • Compute 9's, 10's, 1's, and 2's complements
  • Perform subtraction using r's complement addition
  • Correctly interpret end-carry presence or absence

1. Why Complements?

Complements let digital hardware perform subtraction using only an adder plus a complementing step, avoiding a dedicated subtractor circuit. Every base r system defines two complements: the (r − 1)'s complement and the r's complement.

2. (r − 1)'s Complement

Formula: 
(r − 1)'s complement of N = (rⁿ − 1) − N

3. r's Complement

Formula: 
r's complement of N = rⁿ − N = (r−1)'s complement + 1
10's complement of 2389:
  9's complement = 7610
  Add 1          = 7611  ✓

2's complement of 101100:
  1's complement = 010011
  Add 1          = 010100  ✓

4. Subtraction Using r's Complement

To compute M − N: add M to the r's complement of N, then interpret the result.

💡 Rule: End carry present → positive result, discard carry. No end carry → negative result, take complement of result and attach minus sign.

Worked Example 1 — Decimal, M ≥ N

72532 − 13250
10's complement of 13250 = 86750
Sum = 72532 + 86750     = 159282
Discard end carry       → 59282  ✓ (positive)

Worked Example 2 — Decimal, M < N

13250 − 72532
10's complement of 72532 = 27468
Sum = 13250 + 27468     = 40718
No end carry → negative
10's complement of 40718 = 59282
Result = −59282  ✓

Worked Example 3 — Binary

X = 1010100 (84),  Y = 1000011 (67)

X − Y:
  2's complement of Y = 0111101
  Sum = 1010100 + 0111101 = 10010001
  Discard end carry → 0010001 = +17  ✓

Y − X:
  2's complement of X = 0101100
  Sum = 1000011 + 0101100 = 1101111
  No end carry → take 2's complement → 0010001
  Result = −17  ✓

Common Pitfalls

  • The (r−1)'s complement is bit-wise only; r's complement needs an extra "+1" — conflating them is the most common error.
  • End-carry interpretation is counter-intuitive: carry present = positive (discard it); no carry = negative (complement the result).
  • Always decide the digit width n before complementing.
