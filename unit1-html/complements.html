Complements

⏱ 15 min read 📝 Quiz inside 📖 Mano Ch. 1

Digital hardware is built to add — a subtractor circuit is extra silicon nobody wants to pay for. Complements are the trick that makes this possible: they let a computer perform subtraction using nothing but an adder and a simple bit-flip. This page covers what complements are, the two flavors every radix has, and how they turn A − B into an addition problem.

What Are Complements?

The complement of a number is defined relative to the base (radix) of the number system it's written in, and relative to a fixed number of digits. Every radix r system actually defines two complements:

  • The (r − 1)'s complement (also called the diminished radix complement)
  • The r's complement (also called the radix complement)

For an n-digit number N in base r:

(r − 1)'s complement of N = (rⁿ − 1) − N
>
r's complement of N = rⁿ − N = [(r − 1)'s complement of N] + 1

Notice the second formula is just the first one plus 1 — that relationship is the key to computing either complement quickly.

Types of Complements

In the Decimal System (Base 10)

  • 9's complement (the (r−1)'s complement, since r−1 = 9): subtract each digit from 9, or equivalently compute (10ⁿ − 1) − N.
  • 10's complement (the r's complement): take the 9's complement and add 1.

In the Binary System (Base 2)

  • 1's complement: subtract each bit from 1 — in practice this just means flip every bit (0 becomes 1, 1 becomes 0). This is the fastest possible complement to compute in hardware: no arithmetic at all, just inverters.
  • 2's complement: take the 1's complement and add 1. This is the representation used almost universally for signed integers in real computers (covered in depth on the Fixed-Point Representation page).

Worked Examples

Decimal — find the 9's and 10's complement of 546700:

9's complement = 999999 − 546700 = 453299
10's complement = 453299 + 1 = 453300

Binary — find the 1's and 2's complement of 1011001:

1's complement: flip every bit → 0100110
2's complement: 0100110 + 1 = 0100111

Try it yourself: find the 9's and 10's complement of 246700, and the 1's and 2's complement of 1101100. (Answers in the quiz section below.)

Subtraction Using Complements

Here's the payoff: to compute M − N for unsigned n-digit numbers, add M to the r's complement of N instead of subtracting directly.

Subtraction procedure using the r's complement

Two outcomes are possible, and the hardware distinguishes them purely by whether an end carry appears out of the leftmost digit:

  • End carry produced (M ≥ N): discard the carry. What's left is the answer, M − N, already positive.
  • No end carry (M < N): the raw sum equals rⁿ − (N − M). Take the r's complement of that sum, and the answer is the negative of that value.

Worked example — decimal, carry produced:

72532 − 13250: 10's complement of 13250 = 86750. Sum = 72532 + 86750 = 159282. Discard the leading carry → 59282.

Worked example — decimal, no carry:

13250 − 72532: 10's complement of 72532 = 27468. Sum = 13250 + 27468 = 40718. No carry → answer = −(10's complement of 40718) = −59282.

Worked example — binary:

X = 1010100 (84), Y = 1000011 (67). X − Y: 2's complement of Y = 0111101. Sum = 1010100+0111101 = 10010001; discard the carry → 0010001 (+17).

Common Pitfalls

  1. Don't confuse the two complements. The (r−1)'s complement is a pure digit-wise (or bit-wise) operation; the r's complement needs the extra "+1" on top. Mixing these up is the single most common mistake.
  2. The carry logic is backwards from intuition. A carry present means the result is already correct and positive — just drop the carry. No carry means the result is negative, and needs one more complement step to read in normal form.
  3. Fix the digit width first. A complement is only defined relative to a specific number of digits n — padding a number to more digits changes its complement.

Applications of Complements

  • Representing negative numbers: the 2's complement system is how essentially every modern CPU stores signed integers — the leftmost bit doubles as the sign, and normal binary addition handles negative numbers automatically (see Fixed-Point Representation for the full picture).
  • Simplifying arithmetic circuits: because subtraction reduces to addition-plus-complement, a single adder circuit (with some inverters) handles both operations — real silicon savings at scale.
  • Error detection: checksums, used to catch transmission errors, are often computed as the complement of a data sum — if the received data's checksum doesn't match, something got corrupted in transit.

Quick Quiz

  1. Find the 9's and 10's complement of 246700.
  2. Find the 1's and 2's complement of 1101100.
  3. Compute 13250 − 72532 using 10's complement subtraction (you did this above — confirm you get −59282).
  4. If adding M to the r's complement of N produces no end carry, is the true result of M − N positive or negative?
  5. Why is 1's complement ("flip every bit") faster in hardware than 9's complement ("subtract each digit from 9")?
Check your answers
  1. 9's complement: 999999 − 246700 = 753299. 10's complement: 753299 + 1 = 753300.
  2. 1's complement: flip every bit of 1101100 → 0010011. 2's complement: 0010011 + 1 = 0010100.
  3. −59282 (matches the worked example above).
  4. Negative. No end carry means M < N, so the answer is −(r's complement of the sum).
  5. Flipping a bit is a single inverter gate per bit — pure combinational logic with no dependency between bits. Subtracting each digit from 9 in decimal isn't naturally "free" the way bit inversion is; binary's 1's complement is a hardware-level shortcut that decimal digits don't have.
